Question 3.1

How would you make a web page mobile friendly (ie responsive)?

Answer 3.1

There are many out of the box libraries/frameworks that can be used for this purpose:

  • Bootstrap.
  • Materialize.
  • jQuery Mobile.

Otherwise, media queries can be used for, example:

@media only screen and (max-width: 680px){ body{ background-color: red; } }

Question 3.2

What is the benefit of bundling .js scripts into one file?

Answer 3.2

  • Smaller file sizes for the browser to download.
  • Less files for the browser to download.
  • Allows for module-based development.

Question 3.3

What needs to be done to ensure the browser understands Sass styling?

Answer 3.3

Once a Sass file is compiled, it gets converted into a Css file that the browser understands. If the Sass file is linked to the HTML page, the styling will not work. The HTML page needs to link the generated Css file.

Question 3.4

What should be done to ensure brower compatibility with newer flavours of JavaScript like ES6/7

Answer 3.4

  • Make use of transpiling libraries such as Babel (converts new JavaScript into old JavaScript).
  • Do research on what browsers are most commonly used.
  • Use JavaScript frameworks/libraries that's cross browser compatible.
  • Test your code on older/newer browser versions.
